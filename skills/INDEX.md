# Skills系统文档导航

这是SKyLeapLink项目中的Skills元技能系统的完整文档导航。该系统帮助用户将日常工作转化为LLM可执行的智能技能。

---

## 📚 文档结构

```
📁 skills/ 
│
├─ 📖 入门文档
│  ├─ README.md                    # ← 从这里开始！系统总览
│  └─ QUICK_START.md               # ← 30分钟快速上手指南
│
├─ 📚 核心理论
│  ├─ meta-skills.md               # 元技能指导框架 (深度学习)

│
├─ 📋 模板库
│  └─ templates/
│     ├─ basic-skill.md            # 基础技能模板 (简单任务)
│     └─ workflow-skill.md          # 工作流技能模板 (复杂流程)
│
├─ 📌 示例库
│  └─ examples/
│     ├─ weekly-report.md          # 示例1: 周报撰写 (基础Skill)
│     ├─ requirements-review-workflow.md  # 示例2: 需求评审 (工作流)

│
└─ 🔄 工作流编排 (待完成)
   └─ workflows/
      └─ [多个Skill的组合流程]
```

## 🎯 使用指南

### 我是新用户，如何开始？
```
1. 阅读 README.md (5分钟)
   └─ 理解Skills系统的核心概念

2. 阅读 QUICK_START.md (10分钟)
   └─ 学习如何快速定义第一个Skill

3. 选择模板 (templates/ 文件夹)
   └─ basic-skill.md 或 workflow-skill.md

4. 参考示例 (examples/ 文件夹)
   └─ 看看真实的Skill定义是什么样的

5. 定义你的第一个Skill (30分钟)
   └─ 复制模板，填写你的工作内容

6. 提交给LLM (5分钟)
   └─ 让AI帮你完善规则
```

⏱️ **总耗时: 约50分钟，你就有了第一个可运行的Skill！**

## 📖 文档详解

### 1. README.md
**目标**：理解什么是Skills系统及为什么需要它

**包含内容**：
- Skills的核心概念
- 三层架构（日常工作 → Skills定义 → LLM执行）
- 技能分类概述
- 工作流程示意
- 核心价值

**适合**：所有人，尤其是刚接触的人

**阅读时间**：5-10分钟

---

### 2. meta-skills.md
**目标**：学会如何系统地分析和定义高质量的Skills

**包含内容**：
- 元技能的概念和价值
- 自我审视5问法
- 技能定义框架（4个层级 + 4个维度）
- 定义技能的检查清单
- 与LLM的对话最佳方式
- 持续改进循环

**适合**：想要深入理解Skills定义的人

**阅读时间**：20分钟（完整读）或10分钟（重点读）

---

### 3. categories.md
**目标**：找到最值得自动化的工作，并正确分类

**包含内容**：
- 8种工作领域分类（数据分析、内容生成、决策支持等）
- 按复杂度分类（L1-L4）
- 按工作频率分类
- 快速评估工具（50分评估标准）
- 优先级矩阵

**适合**：需要为自己或团队选择自动化目标的人

**阅读时间**：15分钟

---

### 4. QUICK_START.md
**目标**：在30分钟内掌握如何定义和用Skill

**包含内容**：
- 核心概念速览（5分钟）
- 项目结构浏览（5分钟）
- 选择工作的标准（5分钟）
- 2种定义方式（快速版/详细版）
- 与LLM交互步骤
- 质量检查清单
- 行动计划（短/中/长期）
- 常见问题解答

**适合**：时间紧张、想快速上手的人

**阅读时间**：30分钟（包括定义第一个Skill）

---

### 5. BEST_PRACTICES.md
**目标**：学习成功的做法和避免的陷阱

**包含内容**：
- Skill定义的最佳实践（4个原则）
- 与LLM协作的最佳方式
- 自动化的最佳实践
- 4个常见陷阱及解决方案
- 从个人到组织的扩展
- 进阶技巧

**适合**：已经定义过Skill，想要优化和扩展的人

**阅读时间**：20分钟

## 🎓 模板使用指南

### templates/basic-skill.md
**何时使用**：
- ✅ 工作步骤不超过7步
- ✅ 决策点不超过4个
- ✅ 主要是独立任务
- ✅ 例如：周报、数据导出、邮件分类

**结构**：
```
1. 技能元信息 (YAML格式)
2. 技能概览
3. 输入
4. 处理过程 (步骤序列)
5. 决策规则
6. 输出
7. 背景知识
8. 质量检查清单
9. 更新日志
10. 与LLM的交互提示
```

**填写时间**：30-45分钟

---

### templates/workflow-skill.md
**何时使用**：
- ✅ 工作涉及多个步骤/阶段（6+）
- ✅ 多个人或部门参与
- ✅ 有条件分支或例外流程
- ✅ 例如：需求评审流程、采购审批、新员工入职

**结构**：
```
1. 技能元信息
2. 流程概览
3. 流程架构 (全景图 + 角色分工)
4. 流程阶段详细定义 (每个阶段包含进入条件、输入、决策树、步骤、规则、异常处理等)
5. 条件分支和例外流程
6. 时间计划和SLA
7. 系统集成
8. 质量保证
9. 支持资源
10. 与LLM的交互提示
```

**填写时间**：2-3小时（第一次），后续迭代会快很多

## 📚 示例学习指南

- [ ] 理解复杂流程如何分解为多个阶段

- [ ] 看看如何处理多个决策点

**关键借鉴点**：
1. 流程全景图的绘制方法
2. 每个阶段如何详细定义

## 🔄 学习反馈循环

建议的学习和应用流程：

```
┌─ 阅读文档
│  ├─ 了解概念
│  └─ 看示例
│
├─ 定义Skill
│  ├─ 选择模板
│  ├─ 填写内容
│  └─ 自我检查
│
├─ 提交LLM
│  ├─ AI给反馈
│  ├─ 你改进规则
│  └─ 重复2-3次
│
├─ 在生产中运行
│  ├─ AI执行Skill
│  ├─ 你观察结果
│  └─ 记录问题
│
└─ 迭代改进
   ├─ 分析失败原因
   ├─ 更新规则
   └─ 重新测试
   冷...回到 在生产中运行
```

每个循环通常2-4周。Skill的质量会随着循环而不断提升。
