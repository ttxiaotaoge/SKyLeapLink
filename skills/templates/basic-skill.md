# 基础技能模板 (Basic Skill Template)

使用此模板定义一个**单一、独立的工作任务**。适用于相对简单、步骤较少的工作。

---

## 📌 技能元信息

```yaml
技能名称: [清晰的、动词开头的名称]
分类: [数据分析/内容生成/决策支持/流程自动化/知识整理/沟通协调]
复杂度: [L1-简单 / L2-中等 / L3-复杂 / L4-非常复杂]
频率: [日/周/月/季/年]
预计耗时: [实际执行时间，如：20分钟/小时]
优先级: [高/中/低]
所有者: [负责人姓名]
最后更新: [YYYY-MM-DD]
版本: [1.0]
```

## 🎯 技能概览

### 是什么？
用一句话清晰地描述这个技能的目的。

**示例**：整理和汇总一周所有待办事项完成情况，形成周报。

### 为什么需要这个技能？
说明这个技能的价值和必要性。

- 📊 数据驱动:  [解释如何用数据改进](可选)
- ⏰ 时间节省: [说明节省的时间](必填)
- 📈 质量提升: [说明质量如何改进](可选)
- 🎯 战略对齐: [说明如何支持战略目标](可选)

### 成功标准
明确定义什么样的输出算是成功的。

**示例**：
- ✅ 包含本周至少3项主要成果
- ✅ 用指定的格式模板呈现
- ✅ 所有数据来自官方数据源
- ✅ 无typo或格式错误
- ✅ 在每周五下午5点前交付

## ⚙️ 处理过程

### 步骤序列

按顺序列出所有步骤。每个步骤包括：
- 步骤编号和名称
- 具体操作说明
- 工具工具/资源需求
- 预期输入和输出

**步骤 1: [步骤名称]**
- 操作：[具体做什么]
- 使用工具：[需要什么工具]
- 输入：[这一步的输入]
- 输出：[这一步的输出]
- ⏱️ 预计时间：[多长时间]

**步骤 2: [步骤名称]**
- 操作：[具体做什么]
- 使用工具：[需要什么工具]
- 输入：[这一步的输入]
- 输出：[这一步的输出]
- ⏱️ 预计时间：[多长时间]

**示例**：
```
步骤 1: 收集周任务数据
- 操作：从Jira导出本周完成的所有任务
- 使用工具：Jira API/导出功能
- 输入：Jira访问权限、项目ID
- 输出：本周完成的任务列表（包含任务ID、标题、状态）
- ⏱️ 预计时间：5分钟

步骤 2: 整理和分类
- 操作：按照优先级/模块/团队对任务进行分类
- 使用工具：Excel/Google Sheets
- 输入：任务列表
- 输出：分类后的任务统计（按类别）
- ⏱️ 预计时间：10分钟

步骤 3: 提取关键成果
- 操作：从任务列表中识别最重要的3-5项成果
- 使用工具：文字编辑器
- 输入：分类后的任务列表
- 输出：关键成果描述〈每项1-2句话〉
- ⏱️ 预计时间：5分钟
```

## 🧠 决策规则

### 规则集合
列出在这个技能中应用的所有规则。规则应该是**明确的、可验证的**。

**规则类型 1: [类别]**

```
IF [条件A] THEN [行动B]
ELSE IF [条件C] THEN [行动D]
ELSE [默认行动E]
```

**示例**：

```
【优先级判断规则】
IF 任务影响Revenue THEN 标记为Priority-High
ELSE IF 任务影响客户满意度 THEN 标记为Priority-Medium
ELSE IF 任务是内部优化 THEN 标记为Priority-Low
ELSE 标记为Priority-Backlog
```

### 权重和评分标准
如果涉及评分，说明每个维度的权重。

**示例**：
```
【成果价值评分】
= (战略对齐度 × 40%) + (时间投入 × 30%) + (质量水平 × 20%) + (创新度 × 10%)

战略对齐度：
  - 直接支持OKR: 10分
  - 间接支持OKR: 7分
  - 维持现状: 4分
  - 其他: 1分
```

### 异常处理
描述遇到特殊情况时的处理方式。

**示例**：
```
异常1: 某个关键数据源不可用
→ 处理: 使用上一周的数据代替，并在报告中标注说明

异常2: 任务列表比预期多30%
→ 处理: 只选择Priority-High及以上的任务，其余标注为"待后续补充"

异常3: 无法与某个团队取得联系
→ 处理: 使用上一周的对标数据，并发送跟进邮件
```
