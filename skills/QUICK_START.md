# 快速开始指南 - 30分钟上手Skills系统

如果你时间有限，这个指南会在30分钟内让你了解如何使用Skills系统。

---

## ⏱️ 5分钟：理解核心概念

### 什么是Skills？
**Skills**是一个系统，帮助你将日常工作**转化为AI可理解和执行的指令**。

### 为什么需要？
```
问题: 你每天有很多重复工作，浪费时间
方案: 把这些工作清晰地定义下来
结果: AI代理可以帮你自动完成
```

### 三个关键词

| 词 | 含义 | 例子 |
|---|----|------|
| **Skill** | 一个具体的工作任务 | "撰写周报"、"数据分析" |
| **Workflow** | 多个Skill组合的流程 | "需求从提交到上线的完整流程" |
| **Rule** | 执行Skill时的规则 | "周报必须包含3-5项成果" |

## ⏱️ 5分钟：浏览项目结构

打开 `skills/` 文件夹，你会看到：

```
skills/
├─ README.md                    ← 完整说明文档
├─ meta-skills.md               ← 如何定义好的Skill
├─ categories.md                ← Skill分类体系
├─ templates/                   ← Skill模板（复制这些)
│  ├─ basic-skill.md           ← 适合简单工作
│  └─ workflow-skill.md         ← 适合复杂流程
├─ examples/                    ← 真实例子（参考这些)
│  ├─ weekly-report.md         ← 撰写周报的完整定义
│  └─ requirements-review-workflow.md ← 需求评审流程
└─ workflows/                   ← 工作流编排（[待添加])
```

## ⏱️ 10分钟：选择一个你常做的工作

**选择标准**：
- ✅ 你至少一周做一次或以上
- ✅ 每次做花15分钟或以上
- ✅ 做的方式基本固定
- ✅ 最好是你最讨厌的重复工作😄

**好例子**：
- 每天的日报/进度汇报
- 每周的数据统计
- 定期的代码审查
- 邮件分类和回复
- 周报撰写
- 会议记录整理

**不好例子**：
- 每次都完全不同的创意工作
- 需要大量主观判断的工作
- 做不了超过3次（太稀少）

## ⏱️ 10分钟：定义你的第一个Skill

### 方式1：快速版（仅需5分钟）

直接开始填写，不用完美：

```markdown
# 技能名称
我的日常工作是：[简单描述]

## 输入
- [数据来源1]
- [数据来源2]

## 步骤
1. [第一步做什么]
2. [第二步做什么]
3. [第三步做什么]

## 规则
- IF [条件] THEN [行动]
- IF [条件] THEN [行动]

## 输出
[输出是什么样的]

## 成功标准
- [标准1]
- [标准2]
```

### 方式2：详细版（用10分钟）

1. 选择合适的模板：
   - 简单工作 → 复制 `templates/basic-skill.md`
   - 复杂流程 → 复制 `templates/workflow-skill.md`

2. 逐节填写，参考 `examples/` 中的例子

3. 保存为 `examples/[你的技能名].md`

## 💬 3分钟：与LLM交互

**第一步**：打开你喜欢的LLM（ChatGPT、Claude等）

**第二步**：提交你的Skill定义

```
我要教你一个技能："[技能名称]"

[复制整个Skill定义文件内容]

请帮我：
1. 检查是否有逻辑漏洞
2. 补充我可能遗漏的规则
3. 改进不清楚的地方
4. 准备好执行这个任务
```

**第三步**：取得反馈，改进规则

```
LLM: 我发现你的规则中...

你: 很好的建议。改成这样可以吗？[修改后的规则]
```

**第四步**：让LLM试着执行

```
LLM已学会了这个技能。让我们试试：

[输入一个真实的例子]

请根据我教的规则，为我完成这个任务。
```

## 🎯 检查清单：3分钟确保你的Skill质量

提交给LLM之前，快速问自己：

- [ ] 一个从未做过这件事的人，能根据我的说明完成吗？
- [ ] 有没有遗漏什么关键信息？
- [ ] 规则是够清晰，还是还有歧义？
- [ ] 输入和输出的格式是清楚的吗？
- [ ] 异常情况（出错了怎么办）我有考虑吗？

如果都是✅，恭喜！你的Skill已经可以用了。

## 💡 5个常见问题

### Q1: 我的工作很复杂，怎么办？
**A**: 复杂工作通常可以分解为多个简单技能。先定义最小的单位，然后组合起来。

示例：
```
复杂工作: 整月绩效评估
分解为:
  ├─ 收集员工月度成果
  ├─ 按维度评分
  ├─ 撰写评语
  └─ 汇总为公司报告
```

### Q2: 我的规则有很多例外，怎么办？
**A**: 用决策树或条件语句明确表达例外。

示例：
```
IF 延迟超过1周 THEN 自动升级给经理
ELSE IF 延迟3-7天 THEN 发送提醒邮件
ELSE 正常流程继续
```

### Q3: 如何验证Skill是否正确？
**A**: 让LLM用这个Skill处理3-5个真实例子，看是否符合预期。不对的话改进规则。

### Q4: Skill需要多完美才能用？
**A**: **70%就够好了**。先用起来，边用边改。完美主义是敌人。

### Q5: 如果LLM执行得不对怎么办？
**A**: 通常是规则不够清楚。按照这个流程改进：
1. 找出错误的地方
2. 问"LLM为什么会这样做"
3. 更新规则，让这个错误不会再发生
4. 重新测试
