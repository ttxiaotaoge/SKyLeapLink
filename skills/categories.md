# 技能分类体系 (Skills Classification System)

系统化地对你的工作进行分类，帮助你快速识别哪些工作适合交给AI。

## 🏢 按工作领域分类

### 📊 数据分析类技能
这类工作涉及数据收集、处理、分析和洞察生成。

**特征**：
- 有明确的输入数据
- 计算和分析步骤清晰
- 输出通常是数据/报告

**适合AI的程度**：⭐⭐⭐⭐⭐ (最适合)

**示例技能**：
- 日报数据汇总
- 用户行为分析
- 销售趋势预测
- 成本分析报告
- 竞品对标分析

**AI执行方式**：
```
数据源 → 导入/解析 → 按规则分析 → 生成报表/可视化 → 输出
```

---

### 📝 内容生成类技能
创建各种文字、视觉或多媒体内容。

**特征**：
- 有参考标准或模板
- 遵循特定风格或格式
- 原创性要求可高可低

**适合AI的程度**：⭐⭐⭐⭐ (很适合)

**示例技能**：
- 撰写周报/月报
- 邮件回复模板
- 会议记录
- 文档翻译
- 营销文案
- 代码注释生成
- 演讲稿撰写

**AI执行方式**：
```
主题/要点 → 初稿生成 → 格式调整 → 内容审查 → 输出
```

---

### 🎯 决策支持类技能  
帮助做出判断和决策，基于规则或标准。

**特征**：
- 有明确的评价标准
- 可以列出决策因素
- 可以用规则表达逻辑

**适合AI的程度**：⭐⭐⭐⭐ (很适合)

**示例技能**：
- 需求优先级评审
- 候选人初筛
- 缺陷等级分类
- 绩效评分
- 风险评估
- 合规性检查
- 供应商选择

**AI执行方式**：
```
输入候选项 → 对标标准 → 评分计算 → 规则判断 → 建议输出
```

---

### 🔄 流程自动化类技能
协调多个系统或步骤的工作流程。

**特征**：
- 包含多个顺序步骤
- 涉及多个系统或工具
- 有明确的条件分支

**适合AI的程度**：⭐⭐⭐⭐ (很适合)

**示例技能**：
- 合同审批流程
- 假期申请处理
- 新员工入职流程
- Bug生命周期管理
- 采购订单处理
- 项目启动流程
- 发布部署流程

**AI执行方式**：
```
触发条件 → 步骤1 → 步骤2 → 步骤3 → ... → 结果反馈
```

---

### 💡 知识整理类技能
组织、提炼和转化知识。

**特征**：
- 需要理解和提炼
- 有知识来源
- 输出是结构化知识

**适合AI的程度**：⭐⭐⭐⭐ (很适合)

**示例技能**：
- 技术文档编写
- 最佳实践总结
- 问题FAQ编写
- 知识库文章
- 学习资料整理
- 会议要点提取
- 竞品研究报告

**AI执行方式**：
```
源信息 → 理解/分类 → 提炼要点 → 组织结构 → 生成文档
```

---

### 🤝 沟通协调类技能
与他人交互来交换信息或达成共识。

**特征**：
- 通常有固定的流程
- 可以模板化处理部分工作
- 有标准的沟通内容

**适合AI的程度**：⭐⭐⭐ (中等适合)

**示例技能**：
- 定期问卷调查
- 通知发送
- 反馈收集整理
- 会议日程安排
- 状态更新汇报
- 消息初级分类
- 客户咨询初步回复

**AI执行方式**：
```
列表 → 生成个性化消息 → 发送 → 收集回复 → 分类汇总 → 人工审查
```

---

### 🎓 专业技能（需要深度专业知识）
需要特定领域深度知识的工作。

**特征**：
- 需要专业判断力
- 有新颖或复杂的场景
- 风险和责任较大

**适合AI的程度**：⭐⭐ (较不适合，但可作辅助)

**示例技能**：
- 法律咨询意见
- 医疗诊断建议
- 影响重大的战略决策
- 高风险的安全评估
- 创新产品设计思路

**AI执行方式**：
```
AI提供分析和建议 → 人类专家评审 → 专家做最终决定
```

↳ 重点：AI作为助手，而非决策者

## 📈 按复杂度分类

### 🟢 简单技能 (L1)
- 步骤数：1-3步
- 决策点：0-1个
- 例外情况：很少
- 依赖关系：无或很少
- 学习曲线：很快

**示例**：
- 提取邮件中的关键信息
- 将文本转换为特定格式
- 简单的数据查询和输出

**AI适合度**：✅ 最适合

---

### 🟡 中等复杂技能 (L2)
- 步骤数：4-7步
- 决策点：2-4个
- 例外情况：有一些常见的
- 依赖关系：少数依赖
- 学习曲线：中等

**示例**：
- 周报撰写
- 需求优先级评审
- 定期数据分析报告

**AI适合度**：✅ 很适合

---

### 🔴 高度复杂技能 (L3)
- 步骤数：8+步
- 决策点：5+个
- 例外情况：很多
- 依赖关系：多个依赖
- 学习曲线：较长

**示例**：
- 完整的项目启动流程
- 复杂的采购审批流程
- 多维度的绩效评估系统

**AI适合度**：⚠️ 需要分解成多个小技能，或需要人工干预

---

### ⚫ 极度复杂技能 (L4)
- 步骤数：15+步
- 需要高度的创意和判断
- 多个域的知识整合
- 实时反馈和调整

**示例**：
- 新产品战略规划
- 重大组织重组计划
- 创新项目管理

**AI适合度**：⚠️ 应分解成多个L1-L3技能，AI可协助但不能完全自动化

---

## ⏰ 按工作频率分类

### 🔁 高频任务 (日/周)
**优先级**：🔴 最高 - 最值得自动化

每次重复的小时数 × 频率 = 大量节省时间的潜力

**特征**：
- 容易学习规则
- 有充分的优化空间
- 一旦流程定好，能长期受益

**示例**：
- 日报汇总
- 定期数据分析
- 常见问题回复

---

### 📆 中频任务 (月)
**优先级**：🟡 中等

每月完成一次，但如果耗时长，仍值得自动化。

**示例**：
- 月度报告
- 季度审查总结
- 定期审计报告

---

### 📅 低频任务 (季/年)
**优先级**：🟢 较低

除非：
1. 非常耗时（>8小时）
2. 需要高准确度
3. 有预定的重复时间

**示例**：
- 年度计划制定
- 重大审查会议
- 战略规划工作坊

## 🎯 快速评估工具

### "我应该自动化这个技能吗？"

回答以下问题，计算分数：

```
1. 这个工作有清晰的规则吗？
   ☐ 完全没有 (0分) ☐ 部分有 (5分) ☐ 很清晰 (10分)

2. 做这个工作需要多长时间？
   ☐ <15分钟 (2分) ☐ 15-60分钟 (5分) ☐ >60分钟 (10分)

3. 你多长时间做一次这个工作？
   ☐ 一年少于4次 (2分) ☐ 一月1-4次 (5分) ☐ 一周多次 (10分)

4. 这个工作是否重复性强？
   ☐ 每次都不同 (0分) ☐ 有些重复 (5分) ☐ 非常重复 (10分)

5. 这个工作需要创意或专业判断吗？
   ☐ 非常需要 (0分) ☐ 需要一些 (5分) ☐ 很少需要 (10分)

总分：_____ / 50

解读：
• 40-50分：强烈建议自动化 ⭐⭐⭐⭐⭐
• 30-39分：建议自动化 ⭐⭐⭐⭐
• 20-29分：可以考虑自动化 ⭐⭐⭐
• 10-19分：自动化价值一般 ⭐⭐
• 0-9分：不建议自动化 ⭐
```

## 📊 技能优先级矩阵

```
                  时间节省 ↑
                  很多  |
                       |  ▲ 高优先级
                       |  /  \
中               |      /      \
介               | ▲  /          \  ⭐
               |  /              \
               | /                \
少               |                  \  ▲
                |                    ▼
                ├──────────────────────────→
                频率少      频率中      频率高
```

**四象限分析**：

| 象限 | 特征 | 操作 |
|-----|------|------|
| 🔴 左上 | 耗时多<br>频率低 | 考虑自动化<br>（大收益) |
| 🟠 右上 | 耗时多<br>频率高 | 立即自动化<br>（最值得) |
| 🟡 右下 | 耗时少<br>频率高 | 可以自动化<br>（门槛低) |
| 🟢 左下 | 耗时少<br>频率低 | 暂不优先<br>（收益低) |

## 🔄 持续改进

定期评估你的技能分类：
1. **每季度** - 评估新增或删除的工作
2. **执行后** - 如果自动化成功，写入经验总结
3. **遇见问题** - 如果自动化遇见挑战，优化规则

---
